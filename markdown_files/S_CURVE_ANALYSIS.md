# S-Curve Analysis - Financial Progress Tracking

## Overview

The S-Curve is a cumulative cost curve that tracks planned vs actual project spending over time. It's one of the most important project management tools for monitoring financial and schedule performance.

**Key Purpose**: Visualize the project's financial progress and identify variances between planned and actual spending.

---

## What is an S-Curve?

### Definition
An S-Curve is a cumulative cost curve that typically shows:
- **Planned Cumulative Cost** (baseline)
- **Actual Cumulative Cost** (current performance)
- **Invoiced Cumulative Cost** (payment collection)

### Why S-Shaped?
The curve is typically S-shaped because:
1. **Early Phase**: Slow ramp-up (mobilization, setup)
2. **Middle Phase**: Rapid acceleration (peak execution)
3. **Late Phase**: Gradual decline (closeout, final activities)

## Key Metrics Explained

### Schedule Variance (SV)
**Formula**: Actual Cumulative Cost - Planned Cumulative Cost

**SAMCO Project**:
- Planned to Date (Apr-26): EGP 258,975,039
- Actual to Date (Apr-26): EGP 107,893,864
- **Schedule Variance**: EGP -151,081,175 (-58.3%)

**Interpretation**:
- **Negative SV**: Project is behind schedule
- The project has spent only 41.7% of what was planned by Apr-26
- This indicates slower execution than planned

### Cost Performance Index (CPI)
**Formula**: Earned Value / Actual Cost

**Note**: In S-Curve analysis, we use actual spend as proxy for cost performance.

**SAMCO Project**:
- Actual Spend: EGP 107,893,864
- Planned Spend: EGP 258,975,039
- **Efficiency**: 41.7% (spending at 41.7% of planned rate)

### Invoicing Performance
**Formula**: Invoiced Amount / Actual Spend

**SAMCO Project**:
- Invoiced: EGP 143,405,766
- Actual Spend: EGP 107,893,864
- **Invoicing Ratio**: 133.1%

**Interpretation**:
- Project is over-invoicing (invoicing more than actual spend)
- This is positive for cash flow
- Indicates aggressive invoicing strategy

## Root Cause Analysis

### Why is the Project Behind Schedule?

#### Possible Causes:
1. **Mobilization Delays**
   - Slower than planned site setup
   - Equipment delivery delays
   - Labor mobilization issues

2. **Material Delays**
   - Supplier delays (as noted in delay events)
   - Customs/import delays
   - Quality issues requiring rework

3. **Weather/Site Conditions**
   - Seasonal impacts
   - Site access issues
   - Ground conditions

4. **Resource Constraints**
   - Insufficient labor availability
   - Equipment shortages
   - Subcontractor delays

5. **Design/Approval Delays**
   - Design changes
   - Approval processes
   - Rework requirements

### Why is Invoicing Ahead of Spend?

#### Possible Reasons:
1. **Advance Invoicing Strategy**
   - Invoicing for materials on-site
   - Invoicing for mobilization costs
   - Invoicing for preliminary works

2. **Payment Terms**
   - Advance payments for materials
   - Mobilization advances
   - Retention clauses

3. **Contract Terms**
   - Favorable payment terms
   - Milestone-based invoicing
   - Advance payment provisions

## Technical Details

### Data Source
- **File**: `data/s_curve_data.py`
- **Module**: `get_s_curve_data()`, `get_s_curve_summary()`
- **Format**: Monthly cumulative cost data

### Calculations
- **Schedule Variance**: Actual - Planned (at Apr-26)
- **Invoicing Ratio**: Invoiced / Actual × 100%
- **Remaining Budget**: Total Planned - Total Actual
- **Monthly Average**: Total Actual / Number of Months

### Visualization
- **Chart Type**: Line chart (cumulative) + Bar chart (monthly)
- **Framework**: Plotly Express
- **Interactivity**: Hover information, zoom, pan
